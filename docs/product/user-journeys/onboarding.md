# MVP Onboarding Journey

Status: Draft  
Version: MVP-001

## Goal

Help a new user reach the first useful ceremony with minimal setup.

## Entry State

The user opens the responsive web app with no teas, no teaware, and no saved sessions.

## MVP Journey

1. User sees calm empty states for Tea Shelf, Teaware Shelf, and Brew Timer.
2. User adds one tea with required identity and stock fields.
3. User adds one teaware item with category, material, and volume.
4. User starts a brewing session by selecting tea and optional teaware.
5. User follows a multi-steep timer sequence.
6. User saves the session with notes.
7. User can return later and repeat the session.

## Requirements

- Empty states must point to the next useful action without educational article content.
- Onboarding must not require account creation in MVP.
- Onboarding must not introduce AI or academy flows.
- The user must be able to skip optional details and still create useful records.
- The user must be able to skip teaware and run a tea-only session.
- The user must be able to run a custom first session without inventory.

## Success Criteria

- A first-time user can create one tea, one teaware item, and one session.
- The first ceremony can be completed without leaving the app for timer functionality.
- A first-time user can reach first value without account creation or education content.
