# MVP Retention Journey

Status: Approved for Implementation
Version: MVP-001

## Goal

Encourage repeat use by making prior inventory and session records useful.

## Repeat-Use Loop

1. User returns to browse available teas.
2. User filters or favorites to choose a tea.
3. User selects suitable teaware.
4. User repeats a prior session or starts from a saved recipe.
5. User updates remaining stock after the ceremony.
6. User saves notes for future adjustment.

## Retention Drivers

- Accurate remaining quantity.
- Favorites for frequent teas and recipes.
- Session history that makes successful ceremonies repeatable.
- Low-friction timer access.
- Manual stock update prompt after brewing.
- Useful handling of zero-stock teas and archived records.

## Edge Journeys

- Returning with no saved sessions shows a clear empty state.
- Returning with zero-stock favorite teas keeps them visible but unavailable.
- Repeating a session from history opens a new setup and does not overwrite the original session.

## MVP Exclusions

- Push notifications
- Email reminders
- Social sharing
- AI suggestions
- Marketplace prompts
