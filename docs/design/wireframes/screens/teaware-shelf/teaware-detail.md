# Teaware Detail Layout

Status: Approved for Implementation
Version: MVP-001
Layout Version: LAYOUT-001

## Purpose

Review teaware details and lifecycle actions.

## Layout

- Header: display name or category, edit action, archive/restore status.
- Metadata: category, material, volume, purpose, supported brewing methods.
- Body: notes and any historical session snapshot warning.
- Actions: "Edit", "Archive"; archived detail uses "Restore".

## Responsive Behavior

- Mobile: metadata and notes stack; actions remain directly after header.
- Tablet/desktop: metadata may sit beside notes.
