# MVP Feature Prioritization Matrix

Status: Draft  
Version: MVP-001

| Feature | Priority | Rationale |
| --- | --- | --- |
| Tea CRUD | Must | Core inventory value |
| Teaware CRUD | Must | Core teaware management value |
| Remaining stock | Must | Solves inventory depletion pain |
| Favorites | Must | Helps frequent selection |
| Basic filtering | Must | Makes collections usable |
| Multi-steep timer | Must | Core ceremony workflow |
| Session history | Must | Enables repeatable practice |
| Repeat session | Must | Required for MVP session-history value |
| Recipe favorite | Won't | Deferred with standalone recipe management |
| Compatibility metadata | Should | Supports teaware choice without AI |
| Import/export | Could | Useful but not required for MVP |
| Advanced tasting notes | Could | Valuable later, not core release blocker |
| Beginner prompts | Could | Helpful but must not become academy content |
| AI assistant | Won't | Deferred to Phase 4 |
| Knowledge Shelf | Won't | Deferred to Phase 3 |
| Authentication and sync | MVP+ | Optional post-MVP capability |
| Monetization | Won't | Deferred to Phase 5 |
| Native mobile | Won't | Deferred to Phase 2+ platform strategy |
| Session archive/restore | Must | Protects history and avoids accidental data loss |
| Custom session | Must | Supports first session and flexible ceremony flow |
