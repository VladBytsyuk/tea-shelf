# Stakeholder Profiles

Status: Approved for Implementation
Version: MVP-001

## Product Manager

- Owns scope control, MVP acceptance, roadmap, and spec approval.
- Protects the MVP from AI, content, social, marketplace, and mobile scope creep.

## Architect / Tech Lead

- Owns technical strategy, data model review, system boundaries, and implementation constraints.
- Selects the stack only after product, UX, technical, and QA specs are approved.

## Designer

- Owns UX flows, visual direction, design system requirements, accessibility, and responsive behavior.
- Ensures Chinese Modern Minimalism is usable, not decorative only.

## Engineer

- Owns implementation planning after spec approval.
- Must trace implementation tasks to approved requirements and acceptance criteria.

## QA

- Owns test strategy, acceptance validation, regression scope, and release readiness evidence.

## DevOps

- Owns environment, deployment, CI/CD, release management, and operational readiness once implementation begins.
