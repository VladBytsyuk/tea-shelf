# Deployment Topology

Status: Draft  
Version: MVP-001

## MVP Direction

The MVP deployment target is not selected yet. The expected topology is a single responsive web application deployed through a simple static or web-app hosting path chosen during implementation planning.

## Requirements

- Build artifact must be versioned for release validation.
- Hosting target must support HTTPS.
- Release build must be smoke-tested after deployment.
- Rollback procedure must be documented once hosting is selected.

## Deferred Topology

- Backend API deployment.
- Cloud database.
- Sync workers.
- AI provider integration.
- Native app stores.
