# MVP Team Coordination Plan

Status: Draft  
Version: MVP-001

## Operating Mode

Use Spec-Driven Development. Implementation cannot begin until product, functional, UX, technical, QA, and risk specs are approved.

## Roles

## Product Manager

- Owns MVP scope, priorities, approval workflow, roadmap, and change control.
- Protects MVP from deferred AI, content, monetization, native mobile, social, and marketplace work.

## Architect

- Owns technical strategy, system boundaries, entity model review, and future extensibility constraints.
- Selects stack after spec approval.

## Designer

- Owns UX flows, design system requirements, interaction patterns, accessibility, and responsive behavior.

## Engineer

- Owns implementation planning and implementation after specs are approved.
- Maps implementation tasks to requirement IDs and acceptance criteria.

## QA

- Owns test strategy, test plan, acceptance validation, and release readiness evidence.

## DevOps

- Owns environment, CI/CD, deployment, observability, and release process once implementation planning starts.

## Handoff Format

Each delegated handoff must include:

- Task Context
- Key Inputs
- Decisions
- Deliverables
- Risks / Blockers
- Verification Status
- Escalation Needed
